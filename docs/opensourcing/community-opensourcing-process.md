# Process

This document gives a brief outline of the process for the community to request that a repository be opensourced on rdkcentral github.  Repositories for opensourcing can be either totally new to rdkcentral github or already hosted in private there.

* Code for opensourcing will usually be presented as a repository.
* Other alteratives are possible e.g. a tarfile.
* New repositories will need to be scanned by the RDKM CMF (Code Management Facility) for open source compliance.  To request a scan, please open a ticket [here](https://jira.rdkcentral.com/jira/issues/?jql=project%20%3D%20GHREPOREQ).
* See [this document](https://wiki.rdkcentral.com/display/CMF/New+Repository+and+Repository+Access+Request+FAQ) for guidance on the contents of the GHREPOREQ ticket, for new repositories.
* For repositories which are already hosted on rdkcentral github, an RDKM CMF scanning step is still required.  To request a scan, please open a ticket [here](https://jira.rdkcentral.com/jira/issues/?jql=project%20%3D%20CMFSUPPORT).
* See the [opensource guidelines document](https://github.com/rdkcentral/docs/wiki/opensource-guidelines) for further details on RDKM CMF requirements for opensourcing.
* It is possible to develop completely in the open from the outset.  An empty repository can be requested via a GHREPOREQ ticket, and will only contain template licensing files.  Pull requests to the repository will be checked for Blackduck and copyright compliance.
